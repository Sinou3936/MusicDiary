# 음악 다이어리 기획안

> 오늘의 감정에 맞는 음악을 기록하는 감성 다이어리 서비스

---

## 서비스 개요

매일 듣는 음악에는 그날의 감정이 담겨 있다.  
음악 다이어리는 감정 태그를 선택하면 어울리는 음악을 추천해주고, YouTube에서 찾은 음악을 일기처럼 기록할 수 있는 서비스다.  
기본적으로 비공개이며, 원할 때 다른 사람과 공유할 수 있다.

---

## 타겟 사용자

- 음악으로 감정을 표현하고 싶은 사람
- 나만의 음악 히스토리를 남기고 싶은 사람
- 다른 사람의 감성 플레이리스트가 궁금한 사람

---

## 핵심 기능

### 1. 감정 태그 기반 음악 추천
- 행복, 설렘, 평온, 우울, 그리움, 신남, 외로움, 위로 등 감정 태그 1~3개 선택
- 선택한 감정 태그를 기반으로 어울리는 음악 추천
- 추천 알고리즘: 같은 태그로 기록된 인기 곡 + YouTube 키워드 검색 결과 혼합

### 2. 음악 다이어리 기록
- YouTube에서 직접 곡을 검색해서 기록에 추가
- 감정 태그 1~3개 선택
- 짧은 메모 작성 (선택)
- 공개/비공개 설정 후 저장

### 3. 내 타임라인
- 내가 기록한 음악을 최신순으로 확인
- 곡 제목, 썸네일, 감정 태그, 메모 표시
- 기록 클릭 시 상세 페이지로 이동 + YouTube 바로 듣기 링크

### 4. 탐색 피드
- 다른 사람의 공개 기록 탐색
- 최신순 정렬
- 감정 태그 필터로 원하는 감성만 탐색
- 무한 스크롤

---

## 사용자 플로우

```
Google 로그인
  └─→ 감정 태그 선택 (1~3개)
        ├─→ 음악 추천 받기
        │     └─→ 추천 목록에서 곡 선택
        └─→ 직접 YouTube 검색
              └─→ 검색 결과에서 곡 선택
                    └─→ 메모 입력 (선택)
                          └─→ 공개/비공개 설정
                                └─→ 저장 → 내 타임라인 확인
```

## 화면 구성

| 화면 | 경로 | 주요 내용 |
|------|------|----------|
| 랜딩/로그인 | `/` | 서비스 소개 + Google 로그인 버튼 |
| 내 다이어리 | `/diary` | 내 음악 기록 타임라인 |
| 새 기록 작성 | `/diary/new` | 감정 태그 선택 + YouTube 검색 + 메모 + 저장 |
| 기록 상세 | `/diary/[id]` | 곡 정보, 감정 태그, 메모, YouTube 바로 듣기 |
| 탐색 | `/explore` | 공개된 다른 사람의 음악 기록 피드 |

---

## 데이터 구조

| 테이블 | 설명 |
|--------|------|
| profiles | 사용자 정보 (Google 로그인 연동) |
| emotion_tags | 감정 태그 (시스템 제공 + 사용자 커스텀) |
| diary_entries | 음악 다이어리 기록 (YouTube ID, 메모, 공개 여부) |
| diary_emotion_tags | 기록 ↔ 감정태그 연결 (다대다) |
| recommendations | 감정 기반 추천 기록 |

## 기술 스택

| 영역 | 기술 |
|------|------|
| Frontend | Next.js 14+ (App Router), TypeScript, Tailwind CSS |
| Backend | Supabase (Auth, PostgreSQL, RLS) |
| 외부 API | YouTube Data API v3 |
| 인증 | Google OAuth (Supabase Auth) |
| 배포 | Vercel (Frontend), Supabase Cloud (Backend) |

---

## 보안 정책

- 모든 DB 테이블 RLS(Row Level Security) 적용
- 본인 기록만 수정/삭제 가능
- 비공개 기록은 본인만 조회 가능
- YouTube API 키는 서버에서만 사용 (클라이언트 노출 없음)
